## Game Path

The absolute path to the active game's installation root. Modrex auto-detects this by scanning known
launcher locations for the selected game.

Click **Browse** to set it manually. Modrex validates the path and identifies your launcher when
you save.

## Launcher

Override the auto-detected launcher. Launcher support is game-specific; see
[Supported games](/docs/games/) for the current matrix. Modrex uses this setting to determine how to
start the active game when you click Launch.

## Launch Options

Command-line arguments appended to every launch for the active game. Add `-fileopenlog` here for
PAYDAY 3 if `.pak` mods aren't loading - most PAYDAY 3 mods require it for Unreal Engine to scan the
mods directory.

## New Mod Installs (Crime Boss)

Controls where new Crime Boss mods install by default: **Automatic** (the official ModKit location,
`CrimeBoss/Mods`) or **Always ask**, which prompts you to choose ModKit or the legacy `~mods` location
before each install. See [Crime Boss](/docs/games/crime-boss#choosing-an-install-target) for the
difference between the two targets and how to move an already-installed mod later.

## Skip warnings

Checkboxes to suppress specific one-time warnings:

- **Skip file-open log warning** - hides the reminder to add `-fileopenlog` if it's already set.
- **Dismissed dependency warnings** - per-mod list of dependency warnings you've acknowledged.

Re-enable any warning by unchecking it here.

## Usage analytics

Modrex may ask whether you want to share anonymous usage analytics. This is optional and can be
changed later from **Settings - Usage analytics**. Analytics are off unless you opt in.

## App data locations

Modrex stores its state and settings in the following locations:

| Platform | Path                |
| -------- | ------------------- |
| Windows  | `%APPDATA%\Modrex\` |
| Linux    | `~/.config/modrex/` |

The mod state file (`.modrex.json`) lives inside the active game's managed mods folder so it travels
with the game directory on dual-boot setups. Older `.pd3mm.json` state files are migrated
automatically. See [Priority and state](/docs/concepts/priority-and-state#state-file) for details.
